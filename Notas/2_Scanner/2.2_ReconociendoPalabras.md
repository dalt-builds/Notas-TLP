# Reconociendo palabras
Reconocer una palabra es simplemente analizar cada caracter y preguntarse si es el esperado, por ejemplo para la palabra `new`

![alt text](/img/image4.png)

El diagrama de la derecha es llamado **diagrama de transiciones**

---

## Diagrama de transiciones

Cada **círculo es un estado**.  
Cada **flecha es una transición:** "si estoy en este estado y leo este carácter, paso al siguiente estado". 
El **doble círculo `((s3))` es el estado de aceptación** — llegué aquí significa que reconocí la palabra.

#### 3 conceptos clave:
- **Estado inicial (s0):** donde empieza siempre el reconocedor
- **Estado de aceptación (s*n*):** llegué aquí = palabra válida reconocida
- **Estado de error (implícito):** cualquier carácter que no tiene flecha definida manda al error

# 2.2.1 Formalizacion de reconocedor

### Automata finito (FA):
Una formalizacion de reconocedores con un conjunto finito de estados, un alfabeto, una funcion de transicion, un estado de inicio, y uno o mas estados de aceptacion.
Procesa un carácter por paso, sin retroceder nunca. Eso garantiza que el scanner corre en O(n) — tiempo lineal en el tamaño del input.

**Un automata finito (FA) es la implementacion matematica de un diagrama de transiciones**

---

## Definicion 5-Tupla:
Un FA se define como `(S, Σ, δ, s0, SA)`. Cinco componentes. Vamos uno por uno con el ejemplo de `new` / `not` / `while`:


**S — el conjunto de estados**
Todos los círculos del diagrama, más un estado de error especial se:
`S = {s0, s1, s2, s3, s4, s5, s6, s7, s8, s9, s10, se}`

**Σ (sigma) — el alfabeto**
El conjunto de todos los caracteres que el reconocedor puede leer. Son simplemente las etiquetas de las flechas:
`Σ = {e, h, i, l, n, o, t, w}`  

**δ (delta) — la función de transición**
Esta es la parte central. `δ(estado, carácter) = nuevo_estado` define exactamente a dónde vas desde cada estado al leer cada carácter.
Por ejemplo:
```
δ(s0, 'n') = s1     // desde s0, leer 'n' → ir a s1
δ(s1, 'e') = s2     // desde s1, leer 'e' → ir a s2
δ(s2, 'w') = s3     // desde s2, leer 'w' → ir a s3 (aceptación)
δ(s1, 'o') = s4     // desde s1, leer 'o' → bifurcación hacia "not"
```

Para cualquier otra combinación no definida explícitamente:
```
δ(si, c) = se       // → estado de error
```

**s0 — el estado inicial**
Siempre `s0`. Donde empieza todo.

**SA — los estados de aceptación**
El conjunto de estados de doble círculo:
```
SA = {s3, s5, s10}
```

**Ejemplos:**

Para la palabra `new`:

```
δ(s0, 'n') = s1
δ(s1, 'e') = s2
δ(s2, 'w') = s3
```

`s3 ∈ SA` → ✓ aceptada.

Para la palabra `nut`:

```
δ(s0, 'n') = s1
δ(s1, 'u') = se    ← 'u' no tiene flecha desde s1
δ(se, 't') = se    ← el error state es absorbente, todo lleva a se
```

`se ∉ SA` → ✗ rechazada.

# Frontend
El frontend de un compilador se encarga de analizar el código fuente para verificar que cumple con las reglas sintácticas y semánticas del lenguaje.
Si el código es correcto, lo transforma en una representación intermedia (IR) que será utilizada por las siguientes fases del compilador.
En caso de encontrar errores, el frontend los reporta al usuario y detiene el proceso de compilación.  

### El Scanner (Analizador lexico)
El scanner se encarga de leer caracteres y lo convierte en una lista de palabras clasificadas llamadas **tokens**. 
Cada palabra sale como un par `(tipo, valor)`. Así el resto del compilador no trabaja con letras sueltas sino con unidades con significado.


**Ejemplo:**
`edad = 2026 - 2004`  

[ IDENTIFICADOR('edad'), OPERADOR('='), NUMERO(2026), OPERADOR('-'), NUMERO(2004) ]  

### El parser (Analizador sintactico)
Toma los tokens y se encarga de verificar que esten en el orden correcto segun las reglas de lenguaje de programacion, por medio de una prueba matematica para validar si el codigo es correcto para el lenguaje

**Ejemplo:**
Asegurar que un paréntesis que abre tenga uno que cierra


### Type Checking (Chequeo de Tipos)
Verifica la consistencia del significado semántica. 

**Ejemplo:**
Se asegura de que no intentes sumar un número con una cadena de texto o llamar a una función con el número equivocado de argumentos

# La respresentacion intermedia (IR)
Una vez que el front end confirma que todo está bien, el compilador no salta directamente al código máquina. Primero genera una representación intermedia (IR), que es como un borrador simplificado del programa.

**Ejemplo:**
Para la expresión `a ← a × 2 × b × c × d`, el IR se ve así:
```
t0 ← a × 2
t1 ← t0 × b
t2 ← t1 × c
t3 ← t2 × d
a  ← t3
```

¿Por qué esto? 
El IR descompone la expresión compleja en pasos simples y explícitos. Además, sobre este IR el compilador puede hacer optimizaciones antes de generar el código final.

---

## Ejemplo completo:
Analicemos qué hace el Front End con la instrucción: `x = a + 2`
- **Scanner:** Descompone el texto en una lista clasificada: (identificador, x), (asignador, =), (identificador, a), (operador, +), (número, 2)  

- **Parser:** Determina que esta secuencia corresponde a una "asignación" válida según las reglas gramaticales del lenguaje

- **Type Checker:**  Busca en su tabla de símbolos si la variable a es de un tipo que permita sumarle un número

- **IR:** Crea una estructura lógica para el Optimizador, como: `t1 ← a + 2` y luego `x ← t1`

---

Resumen visual del flujo

```
Tu código fuente
      ↓
   [Scanner]     → separa caracteres en palabras clasificadas
      ↓
   [Parser]      → verifica que la estructura sea válida
      ↓
 [Type Checker]  → verifica que los tipos sean consistentes
      ↓
  [IR Generator] → crea una versión simplificada y optimizable
      ↓
  Código máquina
```

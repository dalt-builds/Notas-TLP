## El modelo de dos fases:
Los compilacion se divide en dos fases principales para separar el lenguaje fuente del lenguaje maquina:  


#### Front-end: 
Se encarga de **analizar el codigo fuente**.
Consta de tres fases:

1. **Analisis lexico (Scanner):** Divide el codigo en tokens (palabras clave, operadores, identificadores)

2. **Analisis sintactico (Parser):** Verifica que la estructura sea correcta (gramática del lenguaje)  

3. **Analisis semantico:** Verifica que tenga sentido lógico (tipos de datos, variables declaradas, etc.)  

El resultado es una representacion intermedia del codigo (IR) 

![Esquema del modelo de dos fases](/img/image.png)

#### **Back-end:**
Se encarga de generar el código final. Toma la IR y la convierte en lenguaje máquina para un procesador específico. 
Consta de dos fases:

- **Optimización:** Mejora el código para que sea más rápido o use menos memoria
- **Generación de código:** Produce el código máquina final

---

## Representacion intermedia (IR):

- Permite que el compilador optimice el codigo de manera eficiente y en varias pasadas

- Reatatering (Readaptacion): Permite compilar para distintas arquitecturas de procesadores con el mismo frontend

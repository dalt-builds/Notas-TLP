# Introduccion a los compildores

## ¿Que es un compilador?
Un compilador es un programa que toma un código escrito en un "lenguaje fuente" (como C++ o Java) y lo **traduce completamente a un "lenguaje maquina"**, generando un archivo ejecutable que el procesador puede ejecutar directamente.

Para hacerlo debe entender:
- Sintaxis (forma del codigo)
- Semantica (signficado)

## Estructura de un compilador

Un compilador tipico tiene:
- **Front-end:** Analiza el lenguaje fuente
- **Back-end:** Genera el lenguaje maquina
- **Representacion intermedia:** Conecta ambos
- **Optimizador:** Analiza, mejora y reescribe la representacion intermedia

## Compilador vs Interprete

Compilador:
- Traduce todo el programa
- Genera un archivo ejecutable

Interprete:
- Ejecuta el codigo directamente
- No genera un archivo ejecutable


## Pricipios fundamentales de un compilador

1. **Preservar el significado:**
El compilador mantiene la integridad del programa original (no cambia ni modifica su funcionamiento)
<br>
2. **Mejorar el programa**:
Lo hace mas eficiente (mas rapido o mas pequeño)

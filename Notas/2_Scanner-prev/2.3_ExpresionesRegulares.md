# Expresion regular (RE)
Un FA especifica un lenguaje visualmente con un diagrama. Una expresión regular especifica el mismo lenguaje pero con una notación textual compacta.

## Los tres operadores principales:
Todo el poder de las expresiones regulares viene de solo tres operaciones:  

**1. Concatenación — "uno seguido de otro"**
Escribir caracteres juntos significa que deben aparecer en ese orden exacto.
```
new    →  reconoce exactamente la palabra "new"
if     →  reconoce exactamente la palabra "if"
```  

**2. Alternación — "uno u otro" (`|`)**
El símbolo `|` significa "o". Elige entre dos opciones.
```
new | while   →  reconoce "new" o "while"
new | not     →  reconoce "new" o "not"
```
Y puedes usar paréntesis para agrupar, igual que en aritmética:

`new or not = new|not = n(ew|ot)`: 

![alt text](/img/image8.png)

**3. Clausura de Kleene — "cero o más veces" (*)**
Este es el operador que rompe la barrera hacia los lenguajes infinitos. Para cualquier RE `x`, escribimos `x*` para significar "cero o más repeticiones de `x`".
```
a*    →  "", "a", "aa", "aaa", "aaaa", ...  (cualquier cantidad de 'a')
(ab)* →  "", "ab", "abab", "ababab", ...
```
El `*` es lo que captura los **ciclos** del FA en forma textual.

---

## La correspondencia RE ↔ FA

Cada operador de RE tiene su equivalente visual en el FA:

| Operación RE     | En el FA                                      |
|------------------|-----------------------------------------------|
| Concatenación `ab` | Transiciones en secuencia: `s0 → a → s1 → b → s2` |
| Alternación `a\|b` | Flechas paralelas desde el mismo estado       |
| Clausura `a*`     | Ciclo: flecha que regresa al mismo estado     |


---

# 2.3.1 Formalizacion de la notacion
Una expresión regular describe un conjunto de cadenas sobre un alfabeto Σ. A ese conjunto lo llamamos lenguaje `L(r)`.
La definición es **recursiva** — construyes REs complejas a partir de REs simples:

**Caso base:**

- Cualquier carácter `a ∈ Σ` es una RE. Describe el conjunto `{a}` — solo esa cadena.
- `ε` es una RE. Describe `{""}` — el conjunto que contiene solo la cadena vacía.

Casos recursivos — si `r` y `s` ya son REs válidas, entonces también lo son:  

## RE vs Lenguaje que describe

| RE        | Lenguaje que describe                                                                 |
|-----------|----------------------------------------------------------------------------------------|
| `r\|s`  | `L(r) ∪ L(s)` — todo lo de r más todo lo de s                                         |
| `rs`      | `{xy \| x ∈ L(r), y ∈ L(s)}` — concatenar uno de r con uno de s                       |
| `r*`      | `L(r)⁰ ∪ L(r)¹ ∪ L(r)² ∪ ...` — r con sí misma, cero o más veces                   |

---

### La clausura de Kleene, formalmente
La definición matemática del texto es:
```
R* = R⁰ ∪ R¹ ∪ R² ∪ R³ ∪ ...
```
Donde:

- `R⁰ = {ε}` — cero repeticiones = la cadena vacía
- `R¹ = R` — una repetición
- `R² = RR` — concatenar R consigo misma
- Y así infinitamente

---

## Precedencia de operadores
```
Mayor precedencia → (paréntesis)
                       ↓
                   * (clausura)
                       ↓
               concatenación (yuxtaposición)
                       ↓
Menor precedencia →  | (alternación)
```
Entonces `ab*|c` se lee como `(a(b*))|c`, no como `a(b*|c)` ni `(ab)*(|c)`.

# 2.3.2 Ejemplos

**Mapa de REs para tokens típicos:**

```
Identificador  →  [a-zA-Z][a-zA-Z0-9]*
Entero         →  0 | [1-9][0-9]*
Real           →  Entero (ε | . [0-9]*)
Científico     →  Real E (ε|+|-) Entero
String C       →  " (^("|\\n))* "
Comentario //  →  // (^\n)* \n
Comentario /*  →  /* ( ^* | *+ ^/ )* */
Registro r0-31 →  r( [0-2][0-9]? | [4-9] | 3[01]? )
```

`^c` significa "cualquier carácter excepto c".

# 2.3.3 Propiedades de Clausura de las REs

La idea central es simple: **las REs son "cerradas" bajo ciertas operaciones**, lo que significa que combinar REs válidas siempre produce otra RE válida.

---

## ¿Qué significa "cerrado bajo una operación"?

Un conjunto es **cerrado** bajo una operación si aplicar esa operación a elementos del conjunto siempre produce otro elemento del mismo conjunto.

**Ejemplo:**
- Los numeros enteros son cerrados bajo suma - sumar enteros siempre da un entero


**Las REs son cerradas bajo sus tres operaciones fundamentales:**

| Operación | Si `r` y `s` son REs... | El resultado también es RE |
|---|---|---|
| Concatenación | `r` y `s` | `rs` ✓ |
| Alternación | `r` y `s` | `r \| s` ✓ |
| Clausura de Kleene | `r` | `r*` ✓ |

---

## Consecuencia 1: puedes construir el scanner completo con una sola RE

Supón que tienes REs separadas para cada categoría de token:
```
a0 = identificador
a1 = entero
a2 = real
a3 = keyword (if | while | for | ...)
a4 = operador (+ | - | * | / | ...)
```

Por clausura bajo alternación, puedes combinarlas todas en **una sola RE**:
```
a0 | a1 | a2 | a3 | a4 | ...
```

El resultado sigue siendo una RE válida. Y por equivalencia RE ↔ FA, puedes construir **un solo FA** que reconoce todos los tokens del lenguaje de una vez.
Esto es exactamente lo que hacen herramientas como `flex`.

---

## Consecuencia 2: cualquier lenguaje finito es regular

Si tienes un conjunto finito de palabras, siempre puedes escribir una RE:
```
palabra1 | palabra2 | palabra3 | ... | palabraN
```

Por clausura bajo unión, eso es una RE válida. Por lo tanto, **cualquier conjunto finito de palabras es un lenguaje regular**. Esto garantiza que todos los keywords de cualquier lenguaje de programación son describibles con una RE.

---

## Consecuencia 3: el complemento también es regular

Dado un FA completo que reconoce `L`, puedes construir un FA que reconoce
**todo lo que no está en `L`** intercambiando estados de aceptación y no-aceptación:

- Estado aceptador → se vuelve no-aceptador
- Estado no-aceptador → se vuelve aceptador

De ahí viene el operador `^` — `^"` es el complemento de `{"}` con respecto
al alfabeto completo.

---

## El cuadro completo de equivalencias
RE  ──────→  FA    (construcción de Thompson, §2.4)
FA  ──────→  RE    (construcción §2.6)

Como puedes ir en ambas direcciones, los dos formalismos describen **exactamente
la misma clase de lenguajes**: los **lenguajes regulares**.

---

## Lenguajes regulares vs. lenguaje natural

En inglés necesitas diccionario para saber si `arz` es una palabra válida.
En programación, la categoría está determinada **completamente por la forma**:
cualquier cadena que cumpla `[1-9][0-9]*` es un entero, sin excepciones ni contexto.

Esta es una **decisión de diseño deliberada** que simplifica enormemente el scanning.
